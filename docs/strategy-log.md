# 戦略ログ（自律戦略家の意思決定の記録）

このファイルは **Opus 戦略家ルーチン（平日昼）** が、市場調査して発案した実験・採否・理由を
日付つきで追記していく場所。継続性と透明性のため。

## 体制
- **戦略家（Opus・平日10:23 JST）**：市場調査 → 新収益源を発案・設計 → GitHub Issue 化。
  - 実装可能な実験 → ラベル `task` `p1` `experiment`（ビルダーが拾う）。
  - 口座・決済・新ドメイン等 運営者の手作業が要るもの → ラベル `needs-operator`。
- **ビルダー（Sonnet・平日9〜17時 2hごと）**：`task` Issue を 1件ずつ自律実装 → PR → CI緑 → 自動マージ → 公開。

## 方針
- データ駆動・EV重視。盛らない。多くのアイデアは飽和/低利益という前提で筋の良いものだけ選ぶ。
- 今の Astro/Cloudflare 基盤内で出せる実験を優先（流入非依存の有料/差別化も可）。
- 当たりが見えたらハブ配下の新サブドメインへ昇格（その時 `needs-operator` で運営者に設定依頼）。
