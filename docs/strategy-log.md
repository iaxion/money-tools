# 戦略ログ（自律戦略家の意思決定の記録）

このファイルは **Opus 戦略家ルーチン（平日昼）** が、市場調査して発案した実験・採否・理由を
日付つきで追記していく場所。継続性と透明性のため。

## 体制
- **戦略家（Opus・平日10:23 JST）**：市場調査 → 新収益源を発案・設計 → GitHub Issue 化。
  - 実装可能な実験 → ラベル `task` `p1` `experiment`（ビルダーが拾う）。
  - 口座・決済・新ドメイン等 運営者の手作業が要るもの → ラベル `needs-operator`。
- **ビルダー（Sonnet・平日9〜17時 2hごと）**：`task` Issue を 1件ずつ自律実装 → PR → CI緑 → 自動マージ → 公開。

## 方針
- データ駆動・EV重視。盛らない。多くのアイデアは飽和/低利益という前提で筋の良いものだけ選ぶ。
- 今の Astro/Cloudflare 基盤内で出せる実験を優先（流入非依存の有料/差別化も可）。
- 当たりが見えたらハブ配下の新サブドメインへ昇格（その時 `needs-operator` で運営者に設定依頼）。

## 2026-07-02 — 律速は「配信」と結論：note.com「借り物ドメイン権威」集客エンジンを起票（#37/#38）

**問題認識（実測起点）**：`docs/scoreboard.md` の実測で **既存13ツール全体の直近7日PV=0**。ビーコン生存・計測正常＝**本当に流入ゼロ**。独立クリティックも前回「律速はツール数ではなくインデックス/ランクするか」と指摘済み。技術SEO（sitemap/robots→sitemap/canonical/GSCメタ対応）は実装済みで**配管の不具合ではない**ことを確認。

**調べたこと**：
1. 新規YMYL金融ドメインのSEO現実（2025-2026）。→ 出典複数で確認：「NISA/手取り」等は 国税庁・銀行・証券・高DA金融メディアが席を独占、**新規の匿名ドメインが正面から上位表示するのは数ヶ月〜1年超の grind**。E-E-A-T（専門家監修・運営者権威）が必須。=**0PVは構造問題で、計算ツール増設では動かない**。
2. 権威を"借りる"配信先。→ **note.com はドメインパワー≈96、質の高い記事は投稿から数週間で検索上位**。金融/投資ジャンル実績多数。有料note/メンバーシップで独立収益化も可（メンバーシップ売上 前年比+81%）。
3. 2026 個人メディアの勝ち筋。→ **SEO単独依存を外すマルチチャネル**（ブログ×X拡散×深掘り）が定石。

**判断（EV順）**：
1. **採用（本命・#37 new-project / needs-operator）：note「借り物ドメイン権威」集客エンジン**。自前ドメインが育つ前に note の DA≈96 を借りて数週間で流入を立て、計算ツールへ送客＋note自体を独立収益化。**プロダクト（正確な計算・公式データ・13ツール）は既にあり、足りないのは配信だけ**という認識に基づく。最大のEVは「**需要検証のフィードバックループを1年→数週間に短縮**」する点（noteでも読まれなければ題材の需要が薄いという速く安い判定）。運営者の最小作業＝**無料note アカウント1つ＋週1〜2本を貼る**のみ。
2. **採用（自律先行・#38 task/p1/experiment）：note配信用 記事ドラフト基盤（`docs/note-drafts/`）＋旬の改正3記事の完成原稿**。運営者判断を待たず agent 単独で完了・**CIリスクゼロ（Markdownのみ）**。#37 の活性化エネルギーを下げ（GO後は"貼るだけ"）、NOでも自前サイトの解説記事に流用可＝下振れ小。旬3本＝iDeCo 2026改正／高額療養費 2026上限／年収の壁178万・基礎控除62万。

**方向性の判断**：今回は「新しい計算ツール」を提案**しない**。0PVの実測を前に11個目12個目を足すのは EV が低い（traffic=0 の関数は動かない）。**律速＝配信** に投資先を切り替える。英語pSEO（#15）は方向性は正しいが**さらに競合激烈な英語×新規ドメイン**で、まず日本語×既存の正確な素材で note の借り物権威を検証する方が安く速い（#15 は保留継続）。

**見送り/正直な限界**：note内は Google AdSense 不可・アフィリ規約に制約（要運営者確認）＝note収益は 有料note/メンバーシップ中心、高単価AdSense/アフィリは送客先の自前ツールで取る設計。note記事の被リンク/ブランド言及は自前ドメインSEOの後追い底上げにもなる。有料note収益はフォロワー形成前提で即時ではない点は盛らない。

**申し送り**：#38 は旬3記事の**数値が公式一次情報と一致**するかが最大リスク（YMYL）。クリティックは iDeCo 6.2/7.5万・施行時期、高額療養費の段階施行、基礎控除62万/年収の壁178万の令和8年値を重点監査。次サイクルは、#37 GO後の note 初期反応（数週間で流入/需要が出るか）を scoreboard と並べて所見する。


## 2026-07-01 — 「2026年 税制/年金 改正」タイミング波に乗る iDeCo クラスタを2件起票

**調べたこと**：2026年が施行年となる日本の主要改正で、鮮度の高い新規検索需要が立ち上がっている領域を調査。
- **年収の壁178万円／基礎控除62万円**（令和8年分から。物価スライドで基礎控除+4万＝62万、合計所得489万以下は104万円等）。
- **iDeCo 拠出限度額 引き上げ**（会社員・企業年金なし 月2.3万→**6.2万**、自営 6.8万→**7.5万**。**2026/12施行**・2027/1引落分から）。
- **退職所得控除「5年ルール→10年ルール」**（令和8年施行。iDeCoと退職金の一時金受取を10年空けないと控除が減額＝「知らないと数十万損」高不安ワード）。
- 子ども・子育て支援金（2026/4・健保料率+0.23%）も手取りに影響（既存 tedori/bonus の鮮度対象）。

**判断（EV順）**：
1. **採用（本命・#23 p1 experiment）：iDeCo 節税シミュレーター（tools/ideco）**。理由：当サイトに NISA はあるが **iDeCo 単独ツールが無い（穴埋め）**。2026/12 の限度額引き上げで旬。証券口座リード＝**高単価アフィリ**。計算は掛金の所得控除ベースで単純＝YMYL低リスク＝先に出す安全な本命。NISA/退職金と相互送客しクラスタ強化。
2. **採用（第2弾・#24 p2 experiment）：iDeCo・退職金 出口戦略「10年ルール」税額シミュレーター（tools/deguchi）**。理由：2026施行の“目玉”で高不安・高関心。受取順×時期×年数で税額が変わり**記事では自分のケースに落とせない＝interactiveの勝ち筋**。退職金(taishoku)クラスタ隣接。**ただしYMYL複雑度が高い**ため、v1スコープを「退職所得控除額＋1/2課税の税額＋10年判定の差額」に限定し、**公式で確定できない控除調整式は金額計算に入れず定性警告に落とす**設計で事故回避（自己レッドチーム済み・要クリティック重点監査）。

**方向性の判断**：今回は「全く別の new-project」ではなく、**既存エンジン（正確な計算機の量産）で“2026改正の実在するタイミング波”に乗る**方を選択。英語pSEO（#15）は独立クリティックの指摘どおり無料検証が先で、別の投機的 new-project を重ねると質が薄まり near-term 収益にも寄与しないため今回は見送り。改正×低競合×高単価アフィリ×低構築コストが揃う JP クラスタが最も EV が高いと判断。

**見送り/メモ**：
- **年収の壁178万円/基礎控除62万円** は魅力的だが、専用の新ツールより **既存 tedori 等の鮮度更新（tax.ts の令和8年値反映）** の色が濃い。これは戦略家の新規発案ではなく**鮮度ルーチン（check-freshness）＋ビルダーの範疇**。次回の鮮度レビューで基礎控除62万/104万・年収の壁の令和8年値、子ども子育て支援金0.23%（2026/4）の反映漏れが無いか確認する旨をここに記録（本ログからの申し送り）。


## 2026-06-30 — 社会保険給付クラスタ新設 ＋ 英語pSEOの賭けを運営者に提示

**調べたこと**：AdSenseの地域別RPM（英語圏 US/UK/CA/AU は日本語の数倍）、社会保険給付（高額療養費/失業手当/育休手当）の需要と競合、補助金/助成金まとめサイトの収益性、英語ニッチ計算機のpSEO機会と競合実態。

**判断（EV順）**：
1. **採用（本命・#13 p1 experiment）：高額療養費 自己負担上限シミュレーター（tools/kogaku）**。
   理由：高不安・高関心の恒常需要に加え、**2025/3見送り→2025/12改正決定→2026/8 から2段階で上限引き上げ施行**という鮮度の追い風（今が改正直前で旬）。公式に統合計算ツールがなく座が空いている。医療/がん保険アフィリと高相性。既存パターンで構築コスト極小。現行値で正確に作り改正値は鮮度レビューで反映（推測ハードコードしない）。
2. **採用（第2弾・#14 p2 experiment）：失業給付（基本手当）シミュレーター（tools/shitsugyo）**。
   理由：退職金ツール群と隣接・相互送客。2025年雇用保険改正で旬。公式テーブル準拠＋calc.test.ts安全弁で正確性担保。#13と合わせ「社会保険給付クラスタ」を形成し topical authority を拡張。
3. **運営者へ提示（中長期の賭け・#15 new-project / needs-operator）：英語ニッチ計算機サイト（別ドメイン・pSEO）**。
   理由：英語RPMの倍率 × pSEOスケールで天井は桁違い。チームの強み（正確な計算機の量産）と合致。ただし新規ドメインの英語SEOは6〜12ヶ月の grind＝**確度低・時間大**。near-termは#13/#14が担うので、保留しても歩みは止まらない。ドメイン/Cloudflare/AdSense/新リポ可否の最小作業の判断を運営者に依頼。

**見送り**：補助金/助成金まとめDB。J-Net21/Jマッチ（約3,000件）が既に網羅、案件の開始/終了が頻繁で**鮮度維持コストが自律運用に重く**、YMYL誤りリスクも高い。EV中だが手間で本命に劣ると判断。

**メモ**：ラベル体系（p1/experiment/new-project/needs-operator）は本日の起票で初めて生成（従来は task/p2/p3 のみ存在）。
