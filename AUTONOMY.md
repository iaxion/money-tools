# 自動運用ガイド（夜間・無人でツールを増やすための手順書）

このファイルは「スケジュール実行のエージェント」や次回以降の作業セッションが、
**毎回ここを読んで1つずつツールを追加・改善していく**ための作業指示書です。
人間の確認なしで安全に回せるよう、作業手順とガードレールを定義します。

---

## 🤖 1回の自動実行でやること（ランブック）

1. `git pull`（あれば）で最新化する。
2. 下の **バックログ** から、未完(`[ ]`)の最優先項目を **1つ** 選ぶ。
3. 計算ロジックは `src/lib/` に**純粋関数**で実装する。
4. 実装直後に簡易チェックを走らせ、既知の概算値と大きくズレないか確認する：
   ```sh
   node --experimental-strip-types <一時テスト.mjs>
   ```
5. ページを `src/pages/tools/<slug>.astro` に作成する。既存の `tools/tedori.astro` を雛形にし、
   **必ず**以下を踏襲：`Base` レイアウト / `path` / `description` / JSON-LD(BreadcrumbList + FAQPage) /
   「概算についての注意」/ 末尾に内部リンク。
6. `src/pages/index.astro` のツール一覧で該当項目を `ready: true` にする。
7. `npm run build` が**通ること**を確認する（通らなければ直してから次へ）。
8. このファイルのバックログにチェック `[x]` を入れ、コミットする：
   ```sh
   # コミット時の識別子はリポジトリの git config を使う（個人メールを直書きしない）
   git add -A && git commit -m "ツール追加: <名称>"
   ```
   （GitHub 連携済みなら自動デプロイで公開される）

## 🛡 ガードレール（絶対に守る）

- **お金をマイナスにしない**：有料サービスを勝手に契約しない。インフラは無料枠
  （Cloudflare Pages 無料・取得済みドメイン）のみ。課金が必要な施策は実行せず README に提案として書くだけ。
- **日本の法律・規約を順守**：誇大表現・無断転載・スパム的大量投稿・自動アカウント作成はしない。
- **正確性が最優先（YMYL）**：税率・保険料率は推測で書かない。国税庁・協会けんぽ等の**公式値を確認**してから反映。
  不確かな場合は「概算」と明記し、確定情報が取れるまで保守的にする。
- **1実行=1ツール程度**に留め、必ずビルドが通る状態でコミットする（壊れた状態を残さない）。
- 既存ツールの計算結果を変える変更をしたら、簡易チェックで妥当性を再確認する。

## 🔁 起動方法（2通り）

### A. クラウドのスケジュール実行（Mac の電源と無関係・本命）
GitHub 連携後に、Claude Code のスケジュール機能（routine / cron）で
「このリポジトリで AUTONOMY.md のランブックを実行」を毎晩走らせる。
1回の実行が利用上限に達しても、**次回のスケジュール実行で自然に続きから再開**する。

### B. ローカルのループ（Mac を起動したまま回す場合）
Mac を起こし続けてこのセッションで回す場合：
```sh
caffeinate -dimsu &     # スリープ防止（電源接続推奨）
```
その上で `/loop` 等で定期実行する。Mac を閉じると止まるため、無人運用は A を推奨。
